<article role="article" about="/covid-19/academic-continuity/zoom-security-settings" class="node node--type-sf-page node--view-mode-full">

  
    

  
  <div class="node__content">
    
      <div class="field field--name-field-sf-content field--type-entity-reference-revisions field--label-hidden field__items">
              <div class="field__item">  <div class="paragraph paragraph--type--sf-section paragraph--view-mode--default">
            <div class="layout layout--onecol">
    <div  class="layout__region layout__region--content">
        <div class="paragraph paragraph--type--sf-text paragraph--view-mode--default">
          
            <div class="clearfix text-formatted field field--name-field-sf-wysiwyg field--type-text-long field--label-hidden field__item"><p class="lead"> </p>

<article class="align-right media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/zoom_security_settings_590x_0.jpg" width="590" height="406" alt="Devices compatible with Zoom" typeof="foaf:Image" /></div>
      
  </article><blockquote>
<p class="lead"><strong>To help members of the UCLA community manage Zoom security concerns, we recommend the following steps for using Zoom while working and learning remotely.</strong></p>
</blockquote>

<p><a class="btn btn--primary" href="https://ucla.box.com/s/qrx8wntj8fx1decsbdfzkubjnrqu1q5n">Download Zoom Security Settings (PDF) </a></p>

<p><a class="btn btn--secondary" href="https://ucla.box.com/s/5easecnbhke1d2wdzvo7cl26vnov8r2q">Zoom Updates July 2020</a></p>

<p><a class="btn btn--invert" href="https://www.it.ucla.edu/news/update-zoom-oct10">Update to Zoom 5.3.0 Is Required</a></p>

<p class="lead"> </p>

<p class="lead"> </p>

<p> </p>

<p> </p></div>
      
      </div>
  <div class="paragraph paragraph--type--sf-accordion paragraph--view-mode--default">
      
<div class="sf_accordion">
  
    <div class="accordion">
  <button type="button" aria-label="click button to view/hide content" class="accordion__label">STEP 1: Zoom web application v. Zoom desktop application</button>
  <div class="accordion__content" role="region">
    <div class="text-formatted"><p><strong>Zoom web application</strong>:</p><p>-You can create a UCLA Zoom account from the <a href="https://ucla.zoom.us/">Zoom at UCLA</a> website. <br /><em>-Before</em> you sign-in to host a meeting, you must first set-up the meeting and properly adjust your Zoom security settings using the Zoom web applicatio<strong>n.</strong><br />
	-Instructions to set-up a secure Zoom meeting using the web application are below.</p><p><strong>Zoom desktop application</strong>:<br />
	The desktop app can be downloaded to your Mac or Windows computer from the <a href="https://ucla.zoom.us/download#client_4meeting">Zoom at UCLA</a> website. </p><p><strong>Recommendations:</strong></p><p>-For instructors and meeting hosts:<br /><em><strong>    </strong>-<strong>After </strong></em>you have set-up the meeting and adjusted the Zoom security settings using the web application, you are strongly encouraged to then use the Zoom <strong>desktop application </strong>instead to conduct the meeting.<br />
	    -Recording a Zoom lecture or meeting <strong><em>requires </em></strong>the use of the desktop app.<br />
	    -We now allow Cloud Recording and will be encouraging it.  We’re currently testing an integration that will move Cloud Recordings from Zoom to a video repository (Kaltura) that is integrated with CCLE.  <br />
	    -Zoom is also integrated with CCLE, so instructors can also schedule their Zoom sessions from within CCLE.<br />
	-For meeting participants (that is, those <em>not </em>hosting Zoom sessions):<br />
	    -Participants may safely join lectures or meetings using either the desktop application or the web application.</p></div>
  </div>
</div>

  
    <div class="accordion">
  <button type="button" aria-label="click button to view/hide content" class="accordion__label">STEP 2: Checking for Desktop Application Updates</button>
  <div class="accordion__content" role="region">
    <div class="text-formatted"><p>If you haven't already, visit <a href="https://ucla.zoom.us/download#client_4meeting">Zoom at UCLA</a> to download the desktop application.</p><p>If you have already downloaded the Zoom desktop application, you need to ensure that you are running the current version.</p><p>Follow these steps to check for the current version.</p><p>1. Open the Zoom application.</p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/open_the_zoom_application_3.png" width="2160" height="1562" alt="Open the Zoom application" typeof="foaf:Image" /></div>
      
  </article><p><br />
	2. Click on your profile in the upper right-hand corner of the Zoom application.</p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/click_on_profile_4.png" width="1280" height="925" alt="Click on profile in upper right-hand corner" typeof="foaf:Image" /></div>
      
  </article><p><br />
	3. Select "<strong>Check for Updates</strong>" from the dropdown menu.     </p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/select_check_for_updates_3.png" width="524" height="1116" alt="Select check for updates" typeof="foaf:Image" /></div>
      
  </article><p><br />
	4. If a new version is available you will get a notification to update; otherwise, you should see the following.</p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/update_status_will_appear_1.png" width="1132" height="390" alt="Update status will appear" typeof="foaf:Image" /></div>
      
  </article><p> </p></div>
  </div>
</div>

  
    <div class="accordion">
  <button type="button" aria-label="click button to view/hide content" class="accordion__label">STEP 3: Set up a Secure Zoom Meeting (use the web app)</button>
  <div class="accordion__content" role="region">
    <div class="text-formatted"><p><strong>1. Log into the Zoom web application. </strong>You can log-in from <a href="https://ucla.zoom.us/">Zoom at UCLA</a> or at <a href="http://www.ucla.zoom.us">ucla.zoom.us</a>. You will be prompted for your UCLA Logon credentials and Duo authentication.</p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/log_into_zoom_at_ucla.zoom_.us__0.png" width="1437" height="1084" alt="Login to Zoom at ucla.zoom.us" typeof="foaf:Image" /></div>
      
  </article><p><br /><strong>2. Schedule a new meeting.</strong> To schedule your meeting, start by selecting “<strong>Schedule a Meeting</strong>.”</p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/schedule_a_meeting_1.png" width="1884" height="582" alt="Select Schedule a Meeting" typeof="foaf:Image" /></div>
      
  </article><p><br /><strong>3. Naming and setting the date. </strong>The first options you should set are the meeting topic and the schedule for the meeting. If you would like to add a description for the class or meeting, you may, but it is not necessary.</p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/naming_and_setting_a_date.png" width="2278" height="1181" alt="Give the meeting a name and set the date and time" typeof="foaf:Image" /></div>
      
  </article><p><br /><strong>4. Creating a recurring meeting (optional). </strong>If you would like the meeting to be a recurring meeting, check the box next to “<strong>Recurring meeting</strong>” and enter the details for the recurrence.</p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/recurring_meeting_optional_0.png" width="1740" height="866" alt="Use the recurring meeting option if applicable" typeof="foaf:Image" /></div>
      
  </article><p><br /><strong>5. Meeting ID and meeting passcode. </strong>Next to Meeting ID, the button for “<strong>Generate Automatically</strong>” should already be checked. <em>We do not recommend using your Personal Room for general sessions.</em> Next to Security, a “<strong>Passcode</strong>” will be generated for you. You can change the passcode, but we recommend that you keep it as a numeric sequence.</p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/meeting_id_and_password_1.png" width="1317" height="596" alt="Meeting ID and a numeric passcode are required." typeof="foaf:Image" /></div>
      
  </article><p><br /><strong>6. Check video settings and mute participants on entry.</strong> When scheduling the meeting, ensure that video is off for both host and participants. You should also ensure that “<strong>Mute participants on entry</strong>” is <em><strong>checked</strong></em>.</p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/turn_off_video_and_turn_on_mute_participants_on_entry.png" width="1172" height="694" alt="Turn off Video and turn on mute participants on entry." typeof="foaf:Image" /></div>
      
  </article><p> </p><p><strong>7. Alternative Hosts.</strong> Finally, if your meeting has a co-host, add the co-host’s email in the “<strong>Alternative Hosts</strong>” field. You can add multiple co-hosts by entering their email addresses separated by commas (for example: <a href="mailto:joebruin@ucla.edu">joebruin@ucla.edu</a>, <a href="mailto:josiebruin@ucla.edu">josiebruin@ucla.edu</a>).</p><p><strong>Important note: </strong>The host and all co-hosts have the same privileges in meetings. A faculty member or meeting chair must be the host or co-host, even when a teaching assistant or other assistant is creating and managing the Zoom session.</p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/alternative_hosts.png" width="1720" height="212" alt="Enter the email address of any alternative hosts" typeof="foaf:Image" /></div>
      
  </article><p><strong>8. Save your meeting.</strong><strong> </strong>Click “<strong>Save</strong>,” and you will see a confirmation that your meeting has been created.</p><p> </p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/save.png" width="540" height="160" alt="After all settings are entered, click SAVE." typeof="foaf:Image" /></div>
      
  </article><p><strong>9. Send participants the information they need to join your meeting. </strong>You can send participants a link that they can click to join your meeting by highlighting and copying the text next to “Join URL.” If you are using CCLE to share the information, we recommend that you add the join URL to your CCLE site <em>(i.e. Turn editing on &gt; select a section &gt; Section tools &gt; URL.).</em></p><p> </p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/copy_the_url_to_invite_attendees.png" width="2686" height="1104" alt="Copy the URL to invite attendees" typeof="foaf:Image" /></div>
      
  </article><p><strong>10. Copy the invitation. </strong> Alternatively, you can send the text of the complete meeting invitation by clicking on “Copy the invitation” (see image above). When you click “Copy the invitation,” another window will pop up and allow you to copy the invitation details. You can then send the copied text to participants or post them in CCLE</p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/copy_meeting_invitation_0.png" width="1318" height="1282" alt="Copy the meeting invitation and post at CCLE" typeof="foaf:Image" /></div>
      
  </article><p><br /><strong>11. Waiting room options. </strong>Once you have saved your meeting, you must make a change in your profile settings. <em>You only have to change this setting once</em> unless you decide to change it again later. Begin by clicking “<strong>Settings</strong>” on the left side of your screen (i.e., Settings&gt; Security&gt; Waiting Room Options</p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/waiting_room_0.png" width="1888" height="744" alt="Confirm the waiting room is enabled and only hosts and co-hosts may admit participants." typeof="foaf:Image" /></div>
      
  </article><p>This will ensure that any user who is logged in using an account from ucla.zoom.us will automatically join your meeting, and it will keep all other users held in the waiting room until you admit them.</p></div>
  </div>
</div>

  
    <div class="accordion">
  <button type="button" aria-label="click button to view/hide content" class="accordion__label">STEP 4: Run your Zoom meeting securely (use the desktop app)</button>
  <div class="accordion__content" role="region">
    <div class="text-formatted"><p><strong>1. Initiate the meeting. </strong>We recommend that you initiate and host meetings using the Zoom <strong>desktop application</strong> (Windows and Mac users). You must use the Zoom desktop application if you are recording a session.</p><p>a. Start the Zoom desktop app: To use the desktop app, you may be asked to log-in. Select the “<strong>Sign in with SSO</strong>” option to ensure you are using the ucla.zoom.us account.</p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/sign_in_sso_0.png" width="1246" height="828" alt="To login select Sign in SSO" typeof="foaf:Image" /></div>
      
  </article><p><br />
	b. UCLA domain: A dialog box will ask for your company domain, which is "UCLA". If the field is empty, type “ucla” in the box and click “<strong>Continue</strong>.”</p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/sso_domain.png" width="1272" height="864" alt="Enter the domain &quot;UCLA&quot;" typeof="foaf:Image" /></div>
      
  </article><p><br />
	2. Begin the meeting. We recommend that the host begin the meeting a few minutes prior to the scheduled start time to allow for managing participants.</p><p>a. To start the meeting, click on the “<strong>Start</strong>” button beside the meeting name.</p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/start_the_meeting.png" width="1846" height="1284" alt="Click Start next to the meeting name" typeof="foaf:Image" /></div>
      
  </article><p><br />
	b. If the meeting does not appear on the main screen, click on the “<strong>Meetings</strong>” button at the top of the desktop application.</p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/meetings_button.png" width="624" height="90" alt="Select the Meetings button if it doesn't automatically load" typeof="foaf:Image" /></div>
      
  </article><p><br /><strong>3. Mute participants in meeting. </strong>After beginning the meeting, the host should select “<strong>Manage Participants</strong>” at the bottom of the meeting window.</p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/select_manage_participants_4.png" width="2306" height="122" alt="Select manage participants at the bottom of the screen." typeof="foaf:Image" /></div>
      
  </article><p><br />
	a. This will open the <strong>Participants </strong>pane on the right-side of the application window.</p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/participants_pane_2.png" width="628" height="233" alt="Participants pane" typeof="foaf:Image" /></div>
      
  </article><p><br />
	b. Select the “<strong>Mute All</strong>” button.</p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/mute_all_0.png" width="637" height="113" alt="Mute all" typeof="foaf:Image" /></div>
      
  </article><p><br />
	c. A dialog box will pop up. Make sure that the “<strong>Allow participants to unmute themselves</strong>” box is <em><strong>unchecked</strong></em>, and then select “<strong>Continue</strong>.”</p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/uncheck_allow_participants_to_unmute_themselves.png" width="824" height="314" alt="Uncheck allow participants to unmute themselves" typeof="foaf:Image" /></div>
      
  </article><p><br /><strong>4. </strong>Limit Chat. Next, select “<strong>Chat</strong>” at the bottom of the Zoom meeting window. This will open a separate chat window.</p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/chat.png" width="2300" height="126" alt="Select chat at the bottom of the screen." typeof="foaf:Image" /></div>
      
  </article><p><br />
	a. At the lower left-side of the chat window, you will see a box with a “<strong>…</strong>” symbol. Click that box.</p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/click_box_with_ellipsis_._symbol.png" width="636" height="718" alt="Click the ellipsis ..." typeof="foaf:Image" /></div>
      
  </article><p><br />
	b. Make sure “<strong>Host Only</strong>” is checked. This will only allow participants to chat with the host to protect the session from unwanted chatter.</p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/host_only.png" width="452" height="236" alt="Select participant can chat with Host Only" typeof="foaf:Image" /></div>
      
  </article><p><br />
	Once these options have been set, you are ready for your meeting to start.</p></div>
  </div>
</div>

  
    <div class="accordion">
  <button type="button" aria-label="click button to view/hide content" class="accordion__label">STEP 5: Manage the Waiting Room</button>
  <div class="accordion__content" role="region">
    <div class="text-formatted"><p>As participants start to join the meeting, those who are using a ucla.zoom.us account will be placed directly in the meeting.</p><p>Those who are not will be shown as “<strong>waiting</strong>” in the Participants window.<br /><br />
	-To admit someone who is waiting, simply click the “Admit” button.<br />
	-If you do not recognize a participant who is waiting, you can either leave them in the waiting room or click the “Remove” button.</p><p><strong>Remember</strong>: Only hosts or co-hosts can move individuals from the waiting room to the class or meeting.</p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/admitting_guests_2.png" width="733" height="465" alt="Select admit to allow guests in the waiting room to join the meeting" typeof="foaf:Image" /></div>
      
  </article><p> </p></div>
  </div>
</div>

  
    <div class="accordion">
  <button type="button" aria-label="click button to view/hide content" class="accordion__label">STEP 6: Deal with unruly participants</button>
  <div class="accordion__content" role="region">
    <div class="text-formatted"><p><span> </span>If you have a participant who starts their video and displays objectionable content, you can quickly terminate that user’s video.</p><p>1. Open "<strong>Manage Participants." </strong></p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/select_manage_participants_3.png" width="2306" height="122" alt="Select manage participants at the bottom of the screen." typeof="foaf:Image" /></div>
      
  </article><p><br />
	2. Move your cursor to the offender’s name in the "<strong>Participants</strong>" pane. Next to the participant’s name, you will see two buttons. Click on the "<strong>More</strong>" button.</p><article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/more_options.png" width="632" height="250" alt="Use the more dropdown for options to manage individual participants" typeof="foaf:Image" /></div>
      
  </article><div>
	<div><br />
	3. From the drop down menu you can select "<strong>Stop Video"</strong> or, if necessary, you can select "<strong>Remove</strong>."</div>

	<div>
	<p><strong>"Stop Video"</strong> does not remove the participant but does restrict their ability to restart their video. The host can restart that participant’s video if desired.</p>

	<p><strong>"Remove" </strong>will open a dialog box, asking you to confirm the action. If you do, the user will be removed and not be able to rejoin the meeting.</p>
	</div>

	<div>
	<article class="media media--type-sf-image-media-type media--view-mode-default"><div class="field field--name-field-media-image field--type-image field--label-hidden field__item">  <img loading="lazy" src="/sites/default/files/media/images/stop_video_or_remove_a_participant.png" width="254" height="321" alt="From the more dropdown you can stop video or remove an individual participant" typeof="foaf:Image" /></div>
      
  </article></div>
	</div><p> </p></div>
  </div>
</div>

  
    <div class="accordion">
  <button type="button" aria-label="click button to view/hide content" class="accordion__label">STEP 7: Actively monitor participants</button>
  <div class="accordion__content" role="region">
    <div class="text-formatted"><p>These recommendations will significantly reduce any participant's ability to disrupt your meeting. </p><p>As the host, you will need to actively monitor participants and deal with any disruptive behavior.</p><p>One additional step you can take is to turn off participants video as they join, using the steps outlined above for unruly participants. When the host turns off the video, the participant cannot restart the video.</p><p>You should also ensure that “<strong>Mute participants on entry</strong>” and “<strong>Enable waiting room</strong>” are both <em><strong>checked</strong></em>.</p></div>
  </div>
</div>

  
    <div class="accordion">
  <button type="button" aria-label="click button to view/hide content" class="accordion__label">STEP 8: Contact Security</button>
  <div class="accordion__content" role="region">
    <div class="text-formatted"><p>If you are having a meeting and you experience a disruption or an uninvited guest, <strong><a href="mailto:security@ucla.edu">email Security</a></strong> as soon as possible so the matter can be investigated further.</p><p>Additionally, anyone who experiences a bias incident should report it by <strong><a href="https://www.deanofstudents.ucla.edu/Report-an-Incident-of-BIAS">completing a UCLA Incident of Bias Reporting Form</a></strong>.</p></div>
  </div>
</div>

  
    <div class="accordion">
  <button type="button" aria-label="click button to view/hide content" class="accordion__label">Additional Zoom Options</button>
  <div class="accordion__content" role="region">
    <div class="text-formatted"><p>Zoom also offers other options that may be value for meeting hosts, including:</p><p><strong>1. <a href="https://support.zoom.us/hc/en-us/articles/201362603-Host-and-Co-Host-Controls-in-a-Meeting?zcid=1231">Lock the meeting</a></strong>. When you lock a Zoom Meeting after it has started, no new participants can join, even if they have the meeting ID and password (if you have required one).</p><p>-In the meeting, click “<strong>Participants</strong>” at the bottom of your Zoom window.<br />
	-In the Participants pop-up, click the “<strong>Lock Meeting</strong>” button.</p><p><strong>2. </strong><strong><a href="https://support.zoom.us/hc/en-us/articles/201362813-Attendee-On-Hold#h_87d70d90-5b0e-46b8-b7b0-b51e3c9fd699?zcid=1231">Put them on hold</a></strong>. You can put each participant on a temporary hold, including the attendees’ video and audio connections.</p><p>-Click on someone’s video thumbnail and select “<strong>Start Attendee On Hold</strong>” to activate this feature.<br />
	-Click “<strong>Take Off Hold</strong>” in the Participants list if and when you’re ready to have them back.</p></div>
